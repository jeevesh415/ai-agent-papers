# 2026年3月〜4月 Memory論文ニュースレター

## 全体トレンド

2026年1〜2月のMemory研究は「メモリ操作自体をエージェントの行動空間として扱う」「平坦なベクトル検索から論理マップ・木・グラフへ」「エピソードを強化学習源として活用する」の3軸で進展した。3〜4月に入り、論点はさらに踏み込んだ次の3方向にシフトしている。

第一に、**「現行のエージェントメモリは記憶ではなく単なるメモ（lookup）に過ぎない」というポジションペーパーの登場**である。神経科学的な海馬／大脳新皮質の二層モデルを引き合いに、現状のRAG・スクラッチパッド・コンテキストウィンドウ管理は「事例ストレージ」しか実装しておらず、抽象ルールが重みに統合される真のメモリには至っていないという問題提起が現れた。これを補強するように、構成主義認識論を組み込んだスキーマ制約型生成（SCG-MEM）や、状況的認知の観点から環境そのものをメモリとして形式化する研究（Artifacts as Memory）など、**「メモリとは何か」を理論側から再定義**する動きが顕在化している。

第二に、**メモリのガバナンスと忘却制御**が独立したサブテーマとして立ち上がった。動的に進化するメモリは意味ドリフト・知識漏洩・メモリ汚染といった新興リスクを伴うことが指摘され、SSGMフレームワーク、Memory Worth指標、規制下のための決定論的射影メモリ（DPM）など、運用適合性を担保する仕組みが提案され始めた。

第三に、**メモリ設計の選択論**が定量化されつつある。長文コンテキストLLM対ファクトベースメモリのコスト分岐点、視覚モダリティを介したコンテキスト圧縮、ドメイン横断でのメモリ転移可能性などが、実証ベースで議論されるようになった。

共通する未解決課題は、(1) **メモリ進化の安全性**（汚染攻撃・意味ドリフト・規制適合）、(2) **抽象度の最適化**（事例特化と汎用転移のトレードオフ）、(3) **「真の記憶」と「メモ」のギャップ**（重みベース統合への道筋）の3点に集約される。

## 1. メモリの構造化アーキテクチャ

### 1.1 グラフ・階層・スキーマ駆動の構造化メモリ

**🔍 このジャンルのトレンドと課題感**
1月のMemoBrain・Structured Episodic Event Memoryで顕在化した「メモリを構造的に推論基盤化する」流れが、4月には**グラフ／スキーマ／階層をどう組み合わせるか**という具体設計の競争段階に入った。共通テーマは、ストリーム的な柔軟性とグラフの堅牢性のバランス、および統合タイミング（いつ意味的にコミットするか）の制御。残課題は、構造化のオーバーヘッドと推論精度の両立、そしてスキーマの自動進化。

---

#### [Apr 2026] "GAM: Hierarchical Graph-based Agentic Memory for LLM Agents" [[paper](https://arxiv.org/abs/2604.12285)]
- **概要**: メモリのエンコーディングと統合を明示的に分離する階層的グラフメモリ。進行中の対話を`event progression graph`に隔離し、意味的シフトが起きた時点で`topic associative network`に統合することで、干渉を最小化しつつ長期一貫性を保持。LoCoMo・LongDialQAでSOTAを更新。
- **差分・新規性**: 「いつ意味的にコミットするか」を明示的なシフト検出に紐づけた点が、ストリーム型の連続更新メモリと離散構造メモリの中間解として独自。

#### [Apr 2026] "StructMem: Structured Memory for Long-Horizon Behavior in LLMs" [[paper](https://arxiv.org/abs/2604.21748)]
- **概要**: 時間推論とマルチホップQAを支えるため、イベントレベルの結合とクロスイベント接続を両立する構造強化型階層メモリ。デュアル視点を時間的にアンカリングし、定期的な意味的統合（semantic consolidation）を行う。LoCoMoで時間推論・マルチホップ性能を改善し、トークン消費・API呼出・実行時間を大幅削減。
- **差分・新規性**: GAMが「いつ統合するか」をシフト駆動で扱うのに対し、本論文は**定期的な統合と時間アンカリング**を採用。フラットメモリと完全グラフのトレードオフへの別解として位置づく。

#### [Apr 2026] "To Know is to Construct: Schema-Constrained Generation for Agent Memory (SCG-MEM)" [[paper](https://arxiv.org/abs/2604.20117)]
- **概要**: 構成主義認識論に着想を得て、LLMデコーディングを「有効なメモリキーのみを生成するよう制約」することで`Structural Hallucination`（存在しないキーの自由生成）を形式的に防ぐスキーマ制約型生成メモリ。同化と調節でスキーマを長期適応させ、Associative Graphで活性化伝播によるマルチホップ推論を実現。LoCoMoで全カテゴリのベースラインを大幅に上回る。
- **差分・新規性**: 「検索ベースの密検索が意味的に似て文脈的に異なるエントリを区別できない」「自由生成は存在しないキーを作る」という2つの典型失敗を、デコーディング側の制約で同時に潰す設計。GAM・StructMemが構造の側で工夫するのに対し、本論文は**生成過程の制約**で問題を解く。

### 1.2 メモリ概念の再定義と理論的問い直し

**🔍 このジャンルのトレンドと課題感**
4月にMemory研究のメタレベル議論が一気に表面化。神経科学・状況的認知・形式モデルから「現状のエージェントメモリは何が足りないか」を問い直すポジションペーパーが立て続けに登場した。残課題は、これら理論的提言を実装に落とすための具体的な設計指針。

---

#### [Apr 2026] "Contextual Agentic Memory is a Memo, Not True Memory" [[paper](https://arxiv.org/abs/2604.27707)]
- **概要**: 現行のエージェントメモリ系（ベクタストア、RAG、スクラッチパッド、コンテキストウィンドウ管理）はメモリを実装しているのではなく**ルックアップ**を実装しているにすぎないと主張するポジションペーパー。神経科学の「高速な海馬の事例ストレージ＋遅い大脳新皮質の重み統合」二層モデルを引き、現行AIエージェントは前者しか実装していないと指摘。
- **差分・新規性**: 3つの構造的問題提起（専門性が獲得できない／構成的タスクで汎化天井に達する／汚染攻撃に脆弱）を整理し、システム構築者・ベンチマーク設計者への前進方向を提示する。1月の`The AI Hippocampus`が認知科学観点での比較に留まったのに対し、本論文は**現行手法の限界を能動的に問題化**する。

#### [Apr 2026] "Artifacts as Memory Beyond the Agent Boundary" [[paper](https://arxiv.org/abs/2604.08756)]
- **概要**: 状況的認知の観点から「環境がエージェントのメモリとして機能的に作用する」ことを強化学習の枠組みで形式化。`アーティファクト`と呼ぶ特定の観測が履歴表現に必要な情報量を削減できることを証明し、エージェントが空間経路を観測すると性能の良いポリシー学習に必要なメモリ量が減ることを実験で示す。
- **差分・新規性**: 内部メモリ機構の改善ではなく、**環境を内部メモリの代替として活用する原理**を形式的に提示する点が他研究にない切り口。「メモリは内部に持つもの」という前提を解体する。

## 2. 経験駆動・自己進化型メモリ

### 2.1 軌跡からのメモリ生成・抽出・転移

**🔍 このジャンルのトレンドと課題感**
平坦なログ蓄積から、軌跡を分解・帰因し**転移可能な抽象化として保存する**潮流が3〜4月にかけて一段と進んだ。同時期にMemory Transfer Learning、CluE、Trajectory-Informed Memoryが揃い、「どの抽象度のメモリが転移するか」を実証的に解析する段階に入っている。残課題は、転移を阻害する低レベル特化メモリのフィルタリング。

---

#### [Mar 2026] "Trajectory-Informed Memory Generation for Self-Improving Agent Systems" [[paper](https://arxiv.org/abs/2603.10600)]
- **概要**: 推論軌跡解析・意思決定属性付け・文脈依存学習生成・適応的メモリ取得の4要素から成る自己改善メモリ生成手法。AppWorldで最大+14.3pt、複雑タスクでは相対149%（+28.5pt）の改善。
- **差分・新規性**: 「どの意思決定が問題だったか」を明示的に属性付けする`Decision Attribution Analyzer`を持つ点。失敗反省を超えた構造的差分を提供する。

#### [Apr 2026] "Self-Evolving LLM Memory Extraction Across Heterogeneous Tasks (CluE)" [[paper](https://arxiv.org/abs/2604.11610)]
- **概要**: パーソナライズ・問題解決・エージェントタスクにまたがる18データセットを再構成したベンチマーク`BEHEMOTH`を提案。訓練例をクラスタ化して各クラスタを独立分析・統合する`CluE`戦略で相対9.04%の改善を達成。
- **差分・新規性**: 「全タスクで支配的な単一静的抽出プロンプトは存在しない」ことを実証した点。1月までのメモリ抽出研究が均質分布を暗黙に前提していたのに対し、**異種性そのものを設計上の中心課題**に据えた。

#### [Apr 2026] "Memory Transfer Learning: How Memories are Transferred Across Domains in Coding Agents" [[paper](https://arxiv.org/abs/2604.14004)]
- **概要**: 異種ドメインから統合されたメモリプールを活用するMTLを提案。6コーディングベンチマーク × 4種類のメモリ表現で評価し、平均+3.7%の性能改善。**転移されるのはタスク固有コードよりもバリデーションルーチン等のメタ知識**であり、抽象度が転移性を決定すると分析。
- **差分・新規性**: 抽象度と転移性の関係を経験的に切り分け、低レベルトレースが負の転移を起こすことを示した点。CluEがプロンプト最適化側で異種性に対応するのに対し、本論文は**メモリ表現の抽象度**を直接の操作対象にする。

---

### 2.2 自律研究・進化的基盤によるメモリ発見

**🔍 このジャンルのトレンドと課題感**
メモリ機構そのものの設計を**自律的な実験ループ・進化動力学・パラメトリック/ノンパラメトリック融合**で発見する研究が登場。1月の`MEMRL`の延長線上にあるが、メモリ機構の探索を自動化する方向が新たに加わった。

---

#### [Apr 2026] "OMNI-SIMPLEMEM: Autoresearch-Guided Discovery of Lifelong Multimodal Agent Memory" [[paper](https://arxiv.org/abs/2604.01007)]
- **概要**: 自律的な研究パイプライン（autoresearch）が問題診断・アーキテクチャ変更・データパイプライン修正を人手介入なしで実施し、マルチモーダルメモリを自動的に改善。LoCoMo +411%（F1: 0.117→0.598）、Mem-Gallery +214%の劇的改善を達成。バグ修正・アーキテクチャ変更・カテゴリ別プロンプトエンジニアリングの単独寄与がハイパラ調整全体を上回り、従来のAutoMLを超えると主張。
- **差分・新規性**: メモリシステム自体の設計探索を**自律研究ループに委ねた**点が独自。マルチモーダルメモリが自律研究パイプラインに適する4特性を提示。

#### [Apr 2026] "Memory Intelligence Agent (MIA)" [[paper](https://arxiv.org/abs/2604.04503)]
- **概要**: Deep Research Agent向けに、Manager-Planner-Executor構成で**パラメトリック／非パラメトリックメモリの双方向変換ループ**を回すフレームワーク。交互強化学習でPlannerとExecutorの協調を強化し、Plannerはテスト時学習で推論を中断せずオンザフライ進化。11ベンチマークで優位性を確認。
- **差分・新規性**: パラメトリックメモリ（Plannerの重み）と非パラメトリックメモリ（圧縮履歴軌跡）を**変換ループで連結**する点。Deep Research領域で、検索コスト増大とメモリ進化の非効率性に同時に対処。

#### [Apr 2026] "Prism: An Evolutionary Memory Substrate for Multi-Agent Open-Ended Discovery" [[paper](https://arxiv.org/abs/2604.19795)]
- **概要**: 階層ファイル永続化／ベクトル意味メモリ／グラフ関係メモリ／進化探索の4パラダイムを単一の決定理論的枠組みで統合。Shannon情報量によるエントロピー閾値階層化、Value-of-Informationによる検索ポリシー、複製子減衰ダイナミクス（`Evolutionary Stable Memory Set`への収束証明）を備える。LOCOMOで+31.2%。
- **差分・新規性**: 情報理論・進化動力学・最適停止理論を組み合わせた**数理的に重い設計**で、形式的収束性を主張する点。経験ベース設計が大半の他研究と明確に区別される。

## 3. メモリのガバナンスと忘却制御

**🔍 このジャンルのトレンドと課題感**
メモリが動的・自己進化的になるほど、**ドリフト・汚染・規制適合性**が顕在化する。3〜4月に登場した3本は、それぞれフレームワーク（SSGM）、運用指標（Memory Worth）、規制下アーキテクチャ（DPM）と異なる粒度で「メモリ運用の安全性」を扱う。共通課題は、ガバナンス機構の汎用化と、性能とのトレードオフ。

---

#### [Mar 2026] "Governing Evolving Memory in LLM Agents: Risks, Mechanisms, and the Stability and Safety Governed Memory (SSGM) Framework" [[paper](https://arxiv.org/abs/2603.11768)]
- **概要**: 動的メモリの新興リスク（メモリ汚染、意味ドリフト、プライバシー脆弱性）を分類した上で、メモリの進化を実行から切り離し、統合前に一貫性検証・時間減衰モデリング・動的アクセス制御を強制する**ガバナンス・アーキテクチャ**を提案。トポロジー誘導型の知識漏洩と反復要約による知識劣化を緩和できることを形式的に分析。
- **差分・新規性**: 既存サーベイが検索効率に集中しているという認識のもと、**メモリ汚染の包括的タクソノミー**を提示。設計より上位の「ガバナンス・パラダイム」を確立しようとする位置づけ。

#### [Apr 2026] "When to Forget: A Memory Governance Primitive" [[paper](https://arxiv.org/abs/2604.12007)]
- **概要**: メモリ品質ガバナンスの運用指標として`Memory Worth (MW)`を提案。メモリの成功/失敗結果との共起を追跡する2カウンタ機構で、定常検索かつ最小探索の下でMWが条件付き成功確率に収束することを証明。10,000エピソード後のスピアマン順位相関ρ=0.89に到達。古いメモリは閾値以下に落ち、専門メモリは高価値を維持。
- **差分・新規性**: メモリ単位あたり**2スカラーカウンタのみ**で実装可能な軽量プリミティブ。SSGMが上位アーキテクチャを論じるのに対し、本論文は**現場で計算可能な指標**として補完関係にある。「因果寄与ではなく結果共起の関連性を測る」という限界を明示している点も誠実。

#### [Apr 2026] "Stateless Decision Memory for Enterprise AI Agents (DPM)" [[paper](https://arxiv.org/abs/2604.20158)]
- **概要**: 規制ドメイン（引受、保険金査定、税務調査）でのエージェント運用は、決定論的リプレイ・監査可能な根拠・マルチテナント分離・水平スケールのためのステートレス性を要求し、**ステートフルメモリは構造的にこれらに違反する**と主張。append-onlyイベントログ＋意思決定時のタスク条件付き投影による`Deterministic Projection Memory`を提案。20倍圧縮時、要約ベースに対し事実精度+0.52・推論一貫性+0.53。決定論性スタディでDPMは1個の非決定論的呼出のみ、要約はN個の連鎖。
- **差分・新規性**: 「ステートフル＝賢い、ステートレス＝劣る」という暗黙の前提を**規制要件側から覆す**点。LongHorizon-BenchでDPMは2 LLM呼出/決定、要約は83-97呼出という極端な効率差も興味深い。

## 4. コスト・モダリティ・運用

### 4.1 ファクトベース vs 長文コンテキスト・視覚活用

**🔍 このジャンルのトレンドと課題感**
コンテキストウィンドウが100k超を当然視できる時代になり、**「メモリシステムを別途持つ意義は何か」**という問いが再燃。4月にはコスト分岐点を定量化する研究と、視覚モダリティで圧縮する研究が登場。

---

#### [Mar 2026] "Beyond the Context Window: A Cost-Performance Analysis of Fact-Based Memory vs. Long-Context LLMs for Persistent Agents" [[paper](https://arxiv.org/abs/2603.04814)]
- **概要**: Mem0ベースのファクトメモリとGPT-5-miniの長文コンテキスト推論を、3つのメモリ中心ベンチマーク（LongMemEval、LoCoMo、PersonaMemv2）で精度・累積APIコスト両軸で比較。長文コンテキストは事実再現率で優位だがpersona一貫性タスクではメモリシステムが競争力を持つ。**100kトークン文脈下では約10ターン以降にメモリシステムが安価になる**ことを定量化。
- **差分・新規性**: prompt cachingを組み込んだコストモデルを構築し、両アーキテクチャがコスト構造の点で根本的に異なることを示した点。本番デプロイの選択基準を提供する実務的価値が高い。

#### [Apr 2026] "OCR-Memory: Optical Context Retrieval for Long-Horizon Agent Memory" [[paper](https://arxiv.org/abs/2604.26622)]
- **概要**: 履歴トラジェクトリを一意の視覚的識別子付き画像にレンダリングし、視覚アンカーで関連領域を選んで対応する逐語テキストを取得する`locate-and-transcribe`パラダイム。自由生成を避けハルシネーションを削減しつつ、任意の長い履歴を高密度に保持する。長期エージェントベンチマークで厳格なコンテキスト制限下で一貫した利得。
- **差分・新規性**: メモリ表現に**視覚モダリティを介した光学的圧縮**を導入する点。テキスト要約とテキスト検索のトレードオフ（情報損失 vs プロンプト負担）を、別モダリティを介して回避する独自の解。

### 4.2 プロアクティブ／プロダクション応用

**🔍 このジャンルのトレンドと課題感**
学術ベンチマーク中心だったメモリ研究に、**実運用とプロアクティブ性**を前提とした要件定義が組み込まれ始めた。LinkedInの本番導入事例とプロアクティブエージェントの一般パラダイムが4月に並んだ。

---

#### [Apr 2026] "PASK: Toward Intent-Aware Proactive Agents with Long-Term Memory" [[paper](https://arxiv.org/abs/2604.08000)]
- **概要**: ストリーミング型プロアクティブAIエージェントの一般パラダイムとして`DD-MM-PAS (Demand Detection, Memory Modeling, Proactive Agent System)`を提案。Pask実装ではDD用ストリーミングIntentFlowモデル、長期MM用のハイブリッドメモリ（workspace/user/global）、PASインフラが閉ループを構成。実世界ベンチマーク`LatentNeeds-Bench`を導入し、レイテンシ制約下でGemini3-Flash相当の性能で深いユーザ意図を識別。
- **差分・新規性**: メモリを**プロアクティブな行動トリガ**として位置づける枠組み。一般的に「問われてから答える」前提のメモリ研究に対し、潜在ニーズの推論と長期メモリを連結する点が独自。

#### [Apr 2026] "Hierarchical Long-Term Semantic Memory for LinkedIn's Hiring Agent (HLTM)" [[paper](https://arxiv.org/abs/2604.26197)]
- **概要**: ノイズの多い縦断的行動データから暗黙・明示シグナルを抽出し、スキーマ整合のメモリツリーに組織化する産業グレードのフレームワーク。スケーラビリティ、低遅延検索、プライバシー制約、クロスドメイン汎化、可観測性の5課題に対応。LinkedIn Hiring Assistantで回答正答率と検索F1を10%以上向上、本番デプロイ済み。
- **差分・新規性**: **本番運用での適合機構**（プライバシー対応保存・透明な来歴・適応機構）を組み込んだ実装報告。学術ベンチでなく実プロダクト指標で示された数少ない事例。
